## 발견된 이슈

### 이슈 1: [Minor] ESLint `react-hooks/set-state-in-effect` 에러 2건

**파일 1**: `pages/DashboardPage.tsx:149`
```
setLoading(true);  // useEffect 본문 내 동기 setState
```

**파일 2**: `contexts/AuthContext.tsx:169`
```
setIsInitializing(false);  // useEffect 본문 내 동기 setState
```

- **내용**: useEffect 본문에서 동기적 setState 호출은 cascading render 유발 가능
- **심각도**: Minor — 빌드/런타임에 영향 없으나 ESLint 규칙 위반
- **비고**: AuthContext 이슈는 기존 코드(오늘 변경 아님). DashboardPage는 오늘 신규 작성
- **영향**: CI에 lint check가 있다면 파이프라인 실패 가능
- **권장 수정**: `useState(true)`로 초기값 설정하여 useEffect 내 `setLoading(true)` 제거

### 이슈 2: [Minor] `ACCESS_TOKEN_KEY` 중복 정의

- **내용**: `'solsqld_access_token'` 문자열이 4개 파일에 각각 하드코딩
  - `utils/api.ts`
  - `contexts/AuthContext.tsx`
  - `api/exams.ts`
  - `pages/SQLPracticePage.tsx`
- **위험**: 값 변경 시 한 곳만 수정하면 불일치 발생
- **심각도**: Minor — 현재는 동일한 값이므로 문제 없음
- **권장**: 향후 리팩토링 시 공통 상수로 추출
